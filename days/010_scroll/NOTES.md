# Day 010 — Interaction · Ascend（R3F）

Week II の3本目。テーマは **スクロール連動（ScrollControls）**。
1つのプロパティをスクロールに紐づけるのではなく、**シーン全体を「スクロールバーがこするタイムライン」** に仕立てた。
1,120本の細い放射状バーを1つの InstancedMesh で螺旋（helix）に巻き、地面から上へ **「塔（spire）」** を組み上げる。
スクロールに連れて **暖色のアクティベーション波面（wavefront）** が塔を昇っていき、通過した下側のバーは静かなボーンの螺旋に落ち着き、まだ届いていない上側のバーは休眠（スケール0）のまま。カメラは波面を追って螺旋状に上昇し、4つのタイポグラフィ章（`<Scroll html>`）がキャプションとして流れる。
**スクロールを戻せば塔は正確に解体される**——スクロールバーはトリガーではなく“トランスポート（再生ヘッド）”。

Day005/007/009 のインスタンシングと Day009 の damped 入力を、**「スクロール＝時間」** へと接続した一歩。

## 狙い
- スクロールを「見た目の主役」にする。1つの 0..1 値（`useScroll().offset`）だけが、カメラの飛行・塔の生成・色の全部を同時に動かす。
- 波面という“いま起きている場所”を常に画面中央に置き、章が変わっても構図が破綻しないよう、**カメラ高さと波面高さを同じ式（`frontOf`）で結ぶ**。
- 配色はボーン一色＋テラコッタ単色アクセント（monaka トーン）、英語エディトリアルな大見出し、余白を大きく。

## 使った技法（今日の“新しい3つ”はすべてスクロール駆動）
1. **useScroll().offset をタイムラインに** — 単一の 0..1 値が、キーフレーム化したカメラ飛行（周回角＋上昇＋ドリー）を毎フレーム補間で駆動する。時計でもポインタでもなく、**スクロール位置が時間**になった。
2. **スクロール波面リビール** — 各バーは正規化高さ u∈[0,1] を持ち、`front = f(offset)` がそれを掃く。波面より下は smoothstep で settle、波面上は Gaussian で暖色 bloom、上は休眠。1つの InstancedMesh に対し、行列と色を毎フレーム波面基準で書き換える（Day009 の per-instance 書込を“時間軸”へ応用）。
3. **`<Scroll html>`** — DOM の章を R3F のスクロールの**内側**に置き、タイポグラフィと3Dを1枚のページとして一緒にスクロールさせる（drei がオーバーレイを portal＋transform する）。

## つまずき / メモ
- 最初、カメラ上昇（camY）と波面（front）を別々に動かしたら、カメラが空を見て塔が隅に小さく映る事故。**両者を `frontOf(offset)` で結び、camY = 波面高さ + 2.6、lookAt = 波面高さ − 1.0** に統一して解決。章ごとに構図が安定した。
- offset 0 で塔が“無”だと1章目が空になるので、`frontOf = 0.10 + offset*0.95` として**開始時点で基部を可視化**、offset 1 で全長が組み上がるよう調整。
- troika `<Text>` は未使用（白画面リスクを回避）。見出しは全部 DOM（`<Scroll html>`）。IBL は手続き Lightformer のみでオフライン安全（Day003/007/008 の教訓）。
