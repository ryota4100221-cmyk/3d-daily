# Day 009 — Interaction · Attend（R3F）

Week II の2本目。テーマは **マウス追従（pointer follow）**。
1つのヒーローをマウスに向けるのではなく、**シーン全体がカーソルに“注意を向ける”** 構図にした。
1,564本の細いロッドを1つの InstancedMesh で敷き、各ロッドが地面上のカーソル位置へ
磁力線のように倒れ込む。近いほど背が伸び、テラコッタに色づき、離れると静かなボーンの原に戻る。
Day005/007 のインスタンシングを **“反応する場”** へ拡張した——Day008 のNOTESで書いた
「1次元の位相波 → 2次元の面」の一歩をここで踏んだ。

## 狙い
- ポインタ入力を「見た目の主役」にする。カーソルの周りだけが立ち上がり暖色に染まることで、
  入力位置が一目でわかる。配色はボーン一色＋テラコッタ単色アクセント（monaka トーン）。
- 場が端で切れず、奥はフォグに溶ける「果てのない原」に見せて没入感を出す。

## 使った技法（今日の“新しい3つ”はすべてポインタ駆動）
1. **RAYCAST → 地面** — `state.pointer` は NDC(−1..1) で奥行きを持たない。毎フレーム
   カメラからレイを飛ばし y=0 平面と交差させ、ロッドが向くべき **実ワールド座標** を得る。
2. **damped follow** — その点を `THREE.MathUtils.damp` で追う。カーソルにスナップせず“重み”を
   持って遅れて追従し、フレームレート非依存（Day008 の delta 習慣を、時計ではなく入力に適用）。
3. **per-instance orient + live color** — 各ロッドが自前でリーン軸（カーソル方向に直交する水平軸）
   と Gaussian 近接度 `exp(-d²/2σ²)` を計算し、**行列と色の両方**を毎フレーム instance バッファへ書く。
   1ドローで 1,564 本が個別に別方向を向く。
- リーン軸 = `cross(up, dir)`。ジオメトリを base へ translate してピボットを根元に置き、
  中心ではなく地面から倒れるようにした（中心ピボットだと下半分が床に沈む）。
- カメラは OrbitControls を外し、ポインタで視差（parallax）＋わずかにカーソル方向を注視。
- 手続きスタジオIBL＋hemisphereライトで高キー・オフライン安全（ランタイムHDRIフェッチ無し）。

## 学び
- 「マウス追従」の質は座標変換で決まる。NDC のまま扱うと平面的、レイで地面に落として初めて
  3D空間の“場”として反応させられる。damp を入力に当てると一気に上質になる。
- 場の反応は「向き・高さ・色」の3属性に分けて近接度1つで駆動すると設計が明快。

## 次に試したいこと（Day 010: スクロール連動へ）
- damp/近接度の考え方を ScrollControls に転用し、スクロール量で場が展開する1ページ体験に。
- カーソル軌跡を残す“余韻”（過去数点を保持して複数の吸引点）で murmuration 風に。
- ロッドをシェーダ変位（Week3）に置き換え、CPUループを GPU へ逃がして本数を1万本規模に。
