# Day 020 — Week 3 · Ink（流体 / 墨流し風 · 移流–拡散の擬似）★★★★

## 狙い
Curriculum #20「流体/墨流し風（ink-lab系・移流・拡散の擬似）」。
Week 3 のシェーダはここまで **すべてステートレス**だった——各フレームが time + pointer から
画像を丸ごと再計算し、前フレームで場がやったことは一切残らない（Day 012 パーティクル、
Day 015–019 のフラグメント/頂点シェーダはすべて one-shot）。
今日の**新技法はその真逆＝本物のステートフルなピンポンFBO流体シミュレーション**。
墨（インク密度）を「移流・拡散・散逸」で場に**記憶**させ、発散ゼロのカール流に折り畳ませて
墨流し（sumi-nagashi）のマーブルを作る。

## 使った技法（新規＝ステートフルなping-pong FBO流体）
- **新技法＝2枚のhalf-float RenderTargetをping-pong**する GPU シミュレーション。
  毎フレーム `read →[ advect · diffuse · dissipate · inject ]→ write` を実行し read/write をスワップ。
  ここまでのステートレス設計（Day 012 が明示的に「速度state/FBO不要」だった）からの明確な前進。
- **セミ・ラグランジュ移流（Stam法）**：各テクセルが `uv - v·dt` を**バックトレース**して
  「流れがどこから来たか」をサンプル。線を描くのではなく密度が引き伸ばされ折り畳まれてフィラメント化。
- **発散ゼロのカールノイズ速度場**：スカラー流れ関数 ψ(=ドリフトする2オクターブ fBm) の
  curl `(∂ψ/∂y, -∂ψ/∂x)` を有限差分で取り、湧き出し/吸い込みのない渦流に（墨が消えず巻くだけ）。
- **拡散＝4近傍平均への微小ブレンド**（0.11）＋**散逸**（密度 ×0.9955 で残る／heat ×0.94 で速く消える）で
  凍結（純移流）と洗い流し（過拡散）の間に置く。
- **2チャンネル状態**：`.r`＝インク密度（ゆっくり散逸）、`.g`＝heat（速く散逸）＝注入直後の「濡れた墨」フラグ。
- **初回フレームの種まきパス**（`uInit`）で fBm マーブルを一発描画し、フレーム1から作品が生きている。以降はカール場が混ぜる。
- **ポインタ＝筆**（Day 009/014/019 系譜）：ガウス・スプラットで墨＋heat を注入し、
  **damp したポインタ速度を流れ場に加算**してインクを実際にストロークへ引きずる（色替えでなく本物の移流ウェイク）。
- **表示パス**：密度→paper→sand→ink ランプ、密度**勾配でエンボス側光**＆フィラメント縁を暗く沈める（濡れ墨の溜まり）、
  heat が濡れ縁を terracotta に温める、grain＋ビネットで monaka 余白。`<Canvas flat>`＋pow(2.2) 線形化。
- half-float RT ＋ 旧GL用 Uint8 フォールバック、noise インライン、troika Text 不使用でオフライン安全・白画面回避。
- `<Canvas orthographic>` にしフルスクリーン・クリップ空間クアッド（表示）＋オフスクリーン正射カメラ（シミュ）で描画。

## 罠 / 学び
- ステートフル化で初めて**RenderTarget を自前 ping-pong** する必要が出た。R3F の宣言的レンダリングとは別に
  `gl.setRenderTarget(write) → gl.render(simScene, simCam) → setRenderTarget(null)` を useFrame 内で手動実行し、
  read/write を useRef で持ち回ってスワップ。表示マテリアルには**書き込み直後**の RT.texture を渡す。
- 種まき（`uInit`）が無いと最初の数秒 真っ黒。周囲ソースだけだと立ち上がりが遅いので初回パスで fBm を敷いた。
- 速度は aspect 空間で等方に計算し、バックトレースの uv ステップへ戻すとき x を uAspect で割る（横長でも渦が丸い）。
- headless_shell（swiftshader）実描画 11s で墨流しのフィラメント・エンボス・余白を確認（ビルド成功≠描画成功）。
