# Day 014 — Week 2 · Capstone "Current"

**難易度 ★★★ / テーマ: Week2まとめ（インタラクションを1つ完成度高く）**

## 狙い
Week2（useFrame / ポインタ追従 / スクロール / 物理 / パーティクル / モーフ）で積んだ
「動きと対話」の技法を、1つの落ち着いた作品に統合する capstone。単なる寄せ集めではなく、
**curl-noise の流れ場に沿ってインスタンスの“向き”を揃える**という新しい一手を軸に、
ポインタが流れを渦へ巻き込む対話を重ねた。紙のトーン・大きな英語タイポ・余白の monaka 系構図。

## 使った技法
- **流れ場にインスタンスの向きを整列（新技法）**: スカラーポテンシャル ψ（依存ゼロの integer-hash 値ノイズ fBm）の
  **curl**（有限差分で `(∂ψ/∂z, -∂ψ/∂x)`）で発散ゼロの2D流れを作り、その方向角に約2,000本のブレードの
  姿勢（`Quaternion.setFromUnitVectors(Yhat, axis)`）を毎フレーム揃える。ブレードはほぼ寝かせて
  流れ方向へ伸ばし、上から見ると curl 場が**風向図（wind-map）**として読める。Day012 の curl noise を
  「点の移流」から「インスタンスの向き」へ転用したのが今回の前進。
- **ポインタ渦（Day009 の応用）**: NDC を y=0 平面へ raycast してワールド座標を取得。近接ガウス
  `w = presence·exp(-d²/2σ²)` で、① 局所の流れ角を `w·3.0` ラジアン回転（渦）② ブレードを伸ばす
  ③ インク→テラコッタに色づけ ④ わずかに起き上がる。カーソルを離すと presence が damp で 0 に戻り、
  渦は自然にほどけて純粋な流れへ復元する（速度バッファ不要のステートレス）。
- **damp によるイージング（Day008）**: presence・ワールド点・カメラ視差をすべて `THREE.MathUtils.damp` で
  時間定数管理し、フレームレート非依存で滑らかに追従／復帰。
- **1ドロー InstancedMesh**: `setMatrixAt` + `setColorAt`、scratch オブジェクト使い回しで毎フレーム GC ゼロ。
  時間項をノイズ座標に足して流れ全体をゆっくり移流（凍らない水のような drift）。
- **オフライン安全**: HDRI フェッチ無し（hemisphere + directional の高キー照明のみ）、troika Text 不使用、
  ノイズはインライン。円盤エッジは smoothstep でフェード、遠方は fog で紙色へ溶かす大気遠近。

## ハマり / 学び（重要）
- **`vertexColors: true` の罠**: InstancedMesh の per-instance カラーは `instanceColor`（`setColorAt`）で
  供給される。マテリアルに `vertexColors: true` を付けると、シェーダが**存在しない geometry の
  `color` 属性**を読もうとして vColor が 0 になり、instanceColor を無視して**全フラグメントが黒**に落ちる。
  今回インク色がほぼ黒だったため気づきにくく、テラコッタが一切出ない状態が続いた。
  **対策: instanceColor を使うなら `vertexColors` は付けない。** ヘッドレス実描画（+ 該当領域のクロップ拡大 +
  window デバッグ書き出し）で切り分けて特定。ビルド成功≠描画成功を改めて確認。
- 上から見る構図では「起き上がる」ブレードは端面＝点になって見えづらい。渦は**寝かせて伸ばし色づけ**する方が
  ずっと読みやすかった（可視性のためのモーション設計）。

## 次に試したいこと（Week3 = シェーダへの助走）
- この curl 整列を**頂点シェーダ**に載せて 5万本規模へ（GPU で向きを計算）。
- 渦の芯に軽い bloom（postprocessing）で「光る流れ」に。
- 流れの速さで**ブレード長も**変調し、速い筋が長く伸びる筆致に。
