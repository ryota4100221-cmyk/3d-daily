# Day 008 — Motion · Breathe（R3F）

Week II のオープナー。テーマは **useFrame基礎 — 回転・浮遊・呼吸**。
一つのオブジェクトに全部を盛るのではなく、モーションを **3つの独立チャンネル** に
分解して、それぞれが単体で読み取れる構図にした。

## 狙い
- モーションを「回転 / 浮遊 / 呼吸」の3系統に分け、各系統を別々にチューニングできる形で見せる。
- 6つのプリミティブを1列に並べ、位相を1つずつずらすことで浮遊・呼吸を **列を伝う波（travelling wave）** にする。
- 配色はほぼニュートラル（ボーン／グレージュ）＋ soft chrome ＋ 単一のマット・テラコッタ。
  「動き」が主役なので色は静かに。monaka トーン。

## 使った技法
- **delta累積の回転**（`spin += delta * speed`）。elapsedTimeから角度を導くDay007と違い、
  リフレッシュレート（60/90/120Hz）に依存しない角速度になる。**今日の“1つ新しい技法”**。
- **3チャンネル分解**：外側group = 浮遊(position.y) + 呼吸(scale)、内側mesh = 回転。
  回転しても浮遊/呼吸は剛体としてかかり、モーションが混ざらない。
- **位相オフセットの正弦コレオグラフィ**：`phase = index * 2π/N` で波が列を伝う。
- **THREE.MathUtils.damp** による登場イージング（settle 0→1）とゆるいスウェイ。
  これもフレームレート非依存。damp は今後のマウス追従/スクロールの土台。
- meshPhysicalMaterial を全ピースで共用し、テラコッタ球だけ clearcoat を載せて質感差を作る。
- 手続きスタジオIBL（Environment + Lightformer）でオフライン安全に高キー描画。ランタイムHDRIフェッチはしない。

## 学び
- 「基礎」テーマこそ、値を time から取るか delta から取るかで質が変わる。delta累積は
  癖にすべき最重要ハビット。
- 動きを channel に分けると、パラメータ表（PIECES配列）だけで全体のリズムを設計できる。

## 次に試したいこと（Day 009: マウス追従へ）
- damp をポインタ追従（視差・カメラ/グループのtilt）に転用する。
- 呼吸に合わせて emissive や envMapIntensity も微妙に脈動させ、「光の呼吸」を足す。
- 位相波を1次元（列）から2次元（面）へ拡張し、Day005/007のinstancingと接続する。
