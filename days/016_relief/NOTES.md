# Day 016 — Week 3 · Shaders "Relief"

**難易度 ★★★★ / テーマ: 頂点シェーダ変位（sin波・ノイズで面を歪ませる）**

## 狙い
Day015 は画面全体を1枚のフラグメントシェーダで描いた「フェイクの立体」だった。今日は
**実ジオメトリの頂点そのものを動かす**。高解像度の `planeGeometry`（220×150 分割）を、
**頂点シェーダの中で** sin 波＋依存ゼロ fBm ノイズの高さ場に沿って法線方向へ変位させ、
遠近のある本物のレリーフ（浮き彫り地形）にする。前日の等高線は、今日は3Dの起伏の上に
そのまま巻き付く（Day015 の callback）。

## 使った技法
- **頂点シェーダ変位（新技法）**: `planeGeometry` の各頂点 `position.z += heightAt(position.xy) * uAmp`。
  高さ場 = `fbm(回転ドリフトするドメイン) - 0.5`（潮のように流れる基層）＋ `sin(x)·cos(y)` の長いうねり
  （カリキュラムの "sin波"）＋ ポインタのガウス隆起。CPU の morph target（Day013）ではなく **GPU で毎頂点**。
- **シェーダ内 analytic normal（今日の"1段上"）**: 変位するとジオメトリのベイク済み法線は表面と合わなくなり、
  そのままだと全面がのっぺり陰なしになる。そこで高さ場を**近傍4点タップ**（±e の x/y）してサンプルし、
  接線 `tx=(2e,0,(hR-hL)·uAmp)`, `ty=(0,2e,(hU-hD)·uAmp)` の**外積で法線を再構築**、`normalMatrix` でワールドへ。
  これで稜線が key ライトを拾い、谷が影に落ちる立体感が出る（変位より**正しくライティングする方が難所**）。
- **手描きライティング（ライト不使用）**: three のライトを使わず、フラグメントで key＋cool fill の 2 灯を dot で自作。
  標高トーン（谷→インク寄りの紙・稜→明るい紙）＋控えめな**フレネルリム**でシルエットを紙から浮かせる。
- **表面上の等高線（Day015 の応用）**: 高さを varying で渡し、`f = vHeight·DENSITY` の整数レベル距離を `fwidth` で
  AA 化して 3D 曲面の上に等高線を描く（5 本ごと主曲線・`extensions.derivatives`）。
- **ポインタ swell（Day009 の系譜）**: レイを**パネルの平面へ ray↔plane で安価にキャスト**（変位前の平面なので
  93k 三角形の intersectObject を避け、`Plane` 1 枚との交差で毎フレーム軽量）→ `worldToLocal` でパネルローカル座標へ。
  その位置を `exp(-d^2/σ^2)` のガウス隆起として高さ場に足し込み、地面が**カーソルへ盛り上がり・テラコッタに温まる**。
  離すと `uHover` が damp で 0 に戻り自然復元（ステートレス）。ポインタ・presence は全て damp でフレームレート非依存。
- **色管理**: `<Canvas flat>`＋`<color attach="background">` の紙色。sRGB パレットを `pow(2.2)` で線形合成し three の
  出力 sRGB 変換で紙色を保つ（Day015 と同じ往復）。troika Text 不使用・フェッチ無しでオフライン安全。

## ハマり / 学び
- 変位した面の法線を放置すると陰影が死ぬ。**analytic normal を頂点で作る**のが立体感の要。有限差分の `e` は
  地形スケール（0.012）に合わせる（大きすぎるとディテールが鈍り、小さすぎるとノイズを拾う）。
- ポインタは**変位後のメッシュ**ではなく**変位前の平面**にキャストするのが正解（安価かつ swell 中心が安定）。
  傾いたパネルなので、法線＝`(0,0,1)` を mesh の quaternion で回して `Plane` を張り、`worldToLocal` でローカル化。
- ビルド成功≠描画成功。ヘッドレス（`headless_shell` バイナリ・`--no-sandbox`・swiftshader）で実描画を1枚撮り、
  変位した起伏・稜線の陰影・表面の等高線が出ることを確認した。
